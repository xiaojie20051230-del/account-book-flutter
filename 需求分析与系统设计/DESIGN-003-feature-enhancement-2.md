# 随手记 — 功能增强设计（二）

DESIGN-003

---

## 1. 概述

基于 GitHub 同类项目调研和用户反馈，对现有 App 进行进一步增强。四个功能按优先级排列。

---

## 2. 功能一览

| 优先级 | 功能 | 工作量 | 说明 |
|--------|------|--------|------|
| **P0** | 数据备份与恢复 | 小 | Hive 文件一键导出/导入 |
| **P1** | 设置页补全收入分类 | 极小 | 预置收入分类也应展示 |
| **P2** | 滑动删除 | 中 | 列表左滑/右滑删除，比长按弹窗优雅 |

---

## 3. P0 — 数据备份与恢复

### 3.1 功能描述

用户可将所有 Hive 数据导出为一个备份文件，也可从备份文件恢复。

### 3.2 痛点

Hive 数据库文件存在 App 私有目录，用户无法直接访问。一旦 App 卸载或数据损坏，所有账单丢失。

### 3.3 技术方案

- **备份**：停止 Hive → 压缩 `*.hive` 文件 → 生成 `.abk`（account book backup）文件 → 调用系统分享
- **恢复**：通过文件选择器选取 `.abk` 文件 → 解压覆盖 Hive 数据库 → 重启 App
- 使用 `share_plus`（已有）和 `file_picker`（新增）实现

### 3.4 交互流程

```
备份: 设置 → 备份数据 → 生成 .abk → 分享到微信/QQ/电脑
恢复: 设置 → 恢复数据 → 文件选择器选 .abk → 覆盖数据 → 重启生效
```

### 3.5 文件清单

| 文件 | 说明 |
|------|------|
| `lib/data/export/backup_manager.dart` | 备份/恢复核心逻辑 |
| `lib/pages/settings/settings_page.dart` | 修改，增加备份/恢复入口 |

## 4. P1 — 设置页补全收入分类

### 4.1 功能描述

当前设置页只展示了**支出**的预置分类（带锁图标），**收入**的预置分类没有展示。应补全。

### 4.2 修改点

在 `_CategoryManager` 中增加收入分类显示区，先展示全部支出预置分类，再展示全部收入预置分类，最后展示自定义分类。

### 4.3 文件清单

| 文件 | 说明 |
|------|------|
| `lib/pages/settings/settings_page.dart` | 修改 `_CategoryManager` |

---

## 5. P2 — 滑动删除

### 5.1 功能描述

账单列表项支持左滑/右滑显示删除按钮，点击直接删除，替代现有长按弹窗流程。

### 5.2 技术方案

- 使用 Flutter 内置的 `Dismissible` 组件包裹每个 `_TransactionTile`
- 滑动时显示红色删除背景 + 垃圾桶图标
- 滑动删除成功后显示 SnackBar，提供"撤销"按钮
- 保留长按弹窗作为备用删除方式

### 5.3 文件清单

| 文件 | 说明 |
|------|------|
| `lib/pages/home/widgets/transaction_list.dart` | 修改 `_TransactionTile`，增加 Dismissible |

---

## 6. P3 — 账单导入（远期规划）

### 6.1 功能描述

导入支付宝/微信导出的 CSV 账单文件，自动解析并批量创建账单。

### 6.2 难点

- 支付宝和微信的 CSV 格式不同
- 日期格式、分类映射需要处理
- 需要引导用户选择平台类型

### 6.3 预留方案

- 新增 `lib/data/import/csv_importer.dart`
- 平台适配策略模式：`AlipayParser`、`WechatParser`
- 暂时不实现，仅预留接口
