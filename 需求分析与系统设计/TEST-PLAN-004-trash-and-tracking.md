## 2. 白盒测试（自动化）

### 2.1 Transaction 模型

| 测试 | 说明 |
|------|------|
| `Transaction.json 包含 updatedAt` | toJson 输出含 updatedAt 字段 |
| `Transaction.fromJson 解析 updatedAt` | 含 updatedAt 时正确解析，不含时为 null |

### 2.2 TrashItem 模型

| 测试 | 说明 |
|------|------|
| `TrashItem.toJson 输出完整` | 所有字段正确序列化 |
| `TrashItem.fromJson 回环一致` | toJson → fromJson 字段不变 |

### 2.3 TransactionProvider 回收站

| 测试 | 说明 |
|------|------|
| `moveToTrash 后列表减少` | 移入回收站后活跃列表长度减 1 |
| `trashItems 包含已删除项` | moveToTrash 后回收站列表长度加 1 |
| `restoreFromTrash 恢复到列表` | 恢复后活跃列表长度加 1，回收站减 1 |
| `permanentlyDelete 从回收站移除` | 永久删除后回收站长度减 1 |
| `update 自动设置 updatedAt` | update 后 updatedAt 不为 null |

## 3. 黑盒测试（手动）

### TC-17：滑动删除至回收站

| # | 步骤 | 预期 |
|---|------|------|
| 1 | 首页向左滑动一条账单 | 账单消失，显示 SnackBar "已删除" |
| 2 | SnackBar 点"撤销" | 账单恢复到列表 |

### TC-18：长按删除至回收站

| # | 步骤 | 预期 |
|---|------|------|
| 1 | 长按一条账单 | 弹出确认弹窗，文案含"移入回收站，7天可恢复" |
| 2 | 点"移入回收站" | 账单消失，SnackBar 可撤销 |
| 3 | 点"取消" | 弹窗关闭，账单不动 |

### TC-19：回收站管理

| # | 步骤 | 预期 |
|---|------|------|
| 1 | 设置 → 回收站 | 显示已删除账单列表，含删除时间 |
| 2 | 点"恢复" | 账单回到首页列表，回收站移除该项 |
| 3 | 点"永久删除" → 确认 | 账单从回收站彻底移除 |
| 4 | 勾选多项 → 批量恢复 | 选中项全部恢复 |
| 5 | 勾选多项 → 批量删除 | 选中项全部永久删除 |
| 6 | 点"清空回收站" → 确认 | 回收站清空 |

### TC-20：修改时间显示

| # | 步骤 | 预期 |
|---|------|------|
| 1 | 首页查看账单列表 | 每项显示 "备注 · 2026-06-02 12:30" 格式时间 |
| 2 | 编辑一条账单并保存 | 返回列表后，时间更新为当前时间 |
