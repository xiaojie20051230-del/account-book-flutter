## 二、分工原则（重要）

### 代码与测试（吕加年全权负责）
- **唯一负责人：吕加年**
- 所有 Flutter 代码（Dart 文件、配置文件、资源文件）由吕加年独立完成
- 功能测试、兼容性测试由吕加年自行完成
- Bug 发现与修复由吕加年独立完成，并记录在 `测试日志.md` 中
- 其他人**不得**直接修改代码文件，**不参与**测试工作

### 文档撰写（四人分工协作）
- **负责人：王进、曹嘉晨、邢子腾、杨玉鑫**
- **实验报告总结部分必须注明每位成员所完成的工作**
- 文档工作包括：
  1. 项目需求分析
  2. 系统结构图绘制
  3. 环境搭建说明
  4. 功能模块截图与说明（吕加年提供截图，文档组整理排版）
  5. 项目总结（总体情况、困难与解决方案、成员分工、课程感想）
- 文档需在代码完成后进行，确保截图和描述与实际代码一致

## 三、测试日志格式

吕加年自行测试时，按以下格式记录到 `测试日志.md`：

```markdown
## 日期：202X-XX-XX

### Bug #1
- **模块**：xxx 页面/功能
- **问题**：具体现象描述
- **修复方案**：如何解决的
- **状态**：已修复 / 待修复

### Bug #2
...
```

## 四、工作流程

```
阶段 1：选题与需求分析（全员讨论）
    ↓
阶段 2：项目设计（吕加年主导架构设计）
    ↓
阶段 3：环境搭建与代码开发（吕加年独立完成）
    ↓
阶段 4：自测与 Bug 修复（吕加年独立完成，记录日志）
    ↓
阶段 5：截图与素材整理（吕加年提供，文档组接收）
    ↓
阶段 6：文档撰写（文档组独立完成）
    ↓
阶段 7：报告审阅与提交（全员确认后提交）
```
