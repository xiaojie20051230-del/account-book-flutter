## 第一步：安装 Android Studio（约 30-60 分钟）

### 1.1 下载

1. 打开浏览器，访问 https://developer.android.com/studio
2. 点击页面上的大按钮 **"Download Android Studio"**
3. 同意条款，下载 **Android Studio Hedgehog | 2024.x.x** 版本（约 1.2 GB）
4. 如果官网下载慢，可以用国内的镜像站

### 1.2 安装

1. 双击下载的 `.exe` 安装文件
2. 点击 **Next** → **Next**
3. 选择安装路径，**建议改到 D 盘**（如 `D:\Android\AndroidStudio`），**不要装 C 盘**
4. 点击 **Install** 开始安装（约 5-10 分钟）
5. 安装完成后勾选 **"Start Android Studio"** → **Finish**

### 1.3 首次启动配置

1. Android Studio 启动后，会弹出 **"Welcome" 向导**
2. 点击 **Next** → 选择 **"Standard"** 标准安装 → **Next**
3. 选择主题（随意，选 Light 或 Dark 都可以）→ **Next**
4. **关键一步：** 在 **SDK 安装路径**界面，把路径改为 **`D:\android-sdk`**（不要默认的 C 盘）
5. 点击 **Finish**，它会自动下载 Android SDK 组件（约 2-3 GB，**预计 20-40 分钟**）
6. **等待右下角进度条走完**，不要中途关闭

### 1.4 验证安装

1. 右下角进度条走完后，点击 **Finish**
2. 会进入 Android Studio 主界面
3. 点击菜单栏 **Help → About**，能看到版本信息即安装成功

### 1.5 常见问题

| 问题 | 解决方法 |
|------|---------|
| 下载 SDK 时卡住不动 | 检查网络，关闭代理/VPN，重试 |
| 提示 "Unable to access Android SDK" | 重启 Android Studio |
| 安装后打不开 | 确保已安装 Microsoft Visual C++ 运行库 |

## 第二步：创建 Android 模拟器（约 15 分钟）

### 2.1 打开 Device Manager

1. 在 Android Studio 主界面，点击右上角图标：**"Device Manager"**（一个手机+小绿点图标）
2. 或者点击菜单栏：**Tools → Device Manager**

### 2.2 创建新设备

1. 点击 **"Create device"**（创建设备）
2. **Category 选择 "Phone"**（手机）
3. 在列表中选 **"Pixel 7"**（或 Pixel 6 也可以）
4. 点击 **Next**

### 2.3 选择系统镜像

1. 点击 **"x86 Images"** 标签
2. 找到 **API 35**（Android 15）那一行
3. 点击右侧的 **"Download"**（约 800 MB，预计 10-15 分钟）
4. 下载完成后，点击 **Next**

⚠️ **如果下载失败**：换个镜像，选 **API 34** 也可以

### 2.4 配置模拟器

1. **AVD Name**：随便起个名字，比如 `MyPhone`
2. **Startup orientation**：选 **Portrait**（竖屏）
3. 其他保持默认
4. 点击 **Finish**

### 2.5 启动模拟器

1. 在 Device Manager 列表中，找到刚创建的设备
2. 点击右侧的绿色 **▶ （播放）按钮**
3. 等待模拟器开机（**第一次启动约 3-5 分钟**）
4. 开机后能看到一个手机屏幕的窗口

**验证模拟器是否正常：** 在模拟器桌面上滑解锁，能看到主屏幕即可。

## 建议：用真机代替模拟器（推荐）

**模拟器的问题：**
- 启动慢、占内存大
- **会在 C 盘占用 5-10 GB 空间**（系统镜像、快照等）
- 运行卡顿

**建议直接用你的 Android 手机。**

### 用真机的好处
- 不用装模拟器，省 5-10 GB C 盘空间
- 运行速度更快
- 截图直接用手机截

### 真机连接步骤

1. **打开手机开发者模式**：
   - 设置 → 关于手机 → 连续点 **7 次"版本号"** → 提示"已开启开发者模式"

2. **打开 USB 调试**：
   - 设置 → 开发者选项 → 打开 **"USB 调试"**

3. **用数据线连接电脑**：
   - 手机上弹出"允许 USB 调试？"→ 勾选"一律允许"→ 点确定
   - 连接模式选 **"传输照片"**（PTP）即可

4. **验证连接**：
   ```
   flutter devices
   ```
   如果显示你的设备型号，就成功了。

5. **运行 App**：
   ```
   flutter run -d <设备ID>
   ```

## ⚠️ 磁盘空间警告

整个开发环境安装完成后，大约会占用 **15-20 GB** 磁盘空间：

| 组件 | 占用空间 |
|------|---------|
| Android Studio | ~1.5 GB |
| Android SDK | ~8 GB |
| Flutter SDK | ~1 GB |
| 模拟器镜像 | ~5 GB（用真机可省掉） |
| Gradle 缓存 | ~1 GB |

### 省空间建议

| 建议 | 说明 |
|------|------|
| ❌ **不要装 C 盘** | Android Studio、Flutter SDK 都装 D 盘 |
| ✅ **用真机代替模拟器** | 省 5 GB 模拟器镜像 |
| ✅ **SDK 装 D 盘** | 安装时选 `D:\android-sdk` |
| ⚠️ **C 盘至少留 20 GB 空闲** | 否则跑不起来 |

## 第三步：安装 Flutter SDK（约 30 分钟）

### 3.1 下载 Flutter SDK

1. 访问 https://docs.flutter.dev/get-started/install/windows
2. 点击 **"Flutter SDK"** 下载链接（约 1 GB，预计 10-20 分钟）
3. 文件名为 `flutter_windows_x.x.x-stable.zip`

### 3.2 解压

**⚠️ 关键：解压路径不能有中文、空格或特殊字符。**

✅ 正确：`D:\flutter`
✅ 正确：`C:\src\flutter`
❌ 错误：`D:\软件\flutter`（含中文）
❌ 错误：`C:\Program Files\flutter`（含空格）

1. 将下载的 `.zip` 文件解压到 `D:\flutter`

### 3.3 配置环境变量

1. 按键盘 **Win + X** → 选 **"系统"**
2. 点击 **"高级系统设置"**
3. 点击 **"环境变量"**
4. 在 **"系统变量"** 中找到 **Path** 变量 → 双击
5. 点击 **"新建"** → 输入 `D:\flutter\bin`
6. 点击 **"新建"** → 输入 `D:\flutter\bin\mingit\cmd`
7. 点击 **"确定"** 关闭所有窗口

### 3.4 验证 Flutter 安装

1. 打开 **CMD**（按 Win + R → 输入 `cmd` → 回车）
2. 输入以下命令：

```
flutter doctor
```

### 3.5 flutter doctor 结果解读

**期望输出（全部打勾）：**

```
Flutter (Channel stable, x.x.x, on Windows) ✓
Android toolchain - develop for Android devices ✓
Android Studio ✓
No issues found!
```

**常见问题：**

| 显示 | 原因 | 解决方法 |
|------|------|---------|
| `Unable to find git in your PATH` | 系统找不到 `where.exe` | 运行下面的命令 |
| `Android toolchain ⚠` | Android Studio 路径没找到 | 检查 Android Studio 是否安装完成 |
| `Android Studio ⚠` | Flutter 没找到 Android Studio | 重启 CMD 再试 |

**如果报 `Unable to find git in your PATH`：**

```
set PATH=C:\Windows\System32;C:\Windows\System32\WindowsPowerShell\v1.0;D:\flutter\bin\mingit\cmd;D:\flutter\bin;%PATH%
```

## 第五步：运行 App（约 10-30 分钟）

### 5.1 打开项目

1. 打开 **CMD**
2. 进入项目目录：

```
cd D:\account_book
```

### 5.2 安装依赖

```
flutter pub get
```

如果成功，会显示：

```
Resolving dependencies...
Downloading packages...
Got dependencies!
```

### 5.3 检查设备

```
flutter devices
```

如果模拟器已启动，应该看到类似输出：

```
1 connected device:
Pixel 7 API 35 (mobile) • emulator-5554 • android-x64 • Android 15 (API 35)
```

### 5.4 运行

```
flutter run
```

**第一次运行特别慢（5-15 分钟）**，因为 Gradle 需要下载依赖：
```
Running Gradle task 'assembleDebug'...（这里会卡很久，等就行）
```

之后就会编译成功并自动安装到模拟器上。

### 5.5 运行失败的处理

| 错误 | 原因 | 解决 |
|------|------|------|
| `Gradle task assembleDebug failed` | 网络问题下载失败 | 等一会重试 `flutter run` |
| `Unable to find git` | 环境变量没配好 | 用上面 set PATH 的命令 |
| `No connected devices` | 模拟器没开 | 检查模拟器是否启动 |
| 编译错误 | 代码问题 | 截图发群里给吕加年 |

### 5.6 运行成功

App 会自动安装并启动，在模拟器上能看到 **随手记** 的首页。

#### 设计思路

```
随手记采用接口驱动 + Repository 模式设计（参见 ADR-001 架构决策记录）。
业务逻辑与数据存储解耦，后端使用 Hive 本地数据库，
状态管理使用 Provider，UI 采用 Material Design 3 设计语言。

前期完成了完整的设计文档体系（DESIGN-001~008），
涵盖 MVP 架构、功能扩展、回收站、附件、密码锁等模块的详细设计。

核心设计原则：
1. 离线优先——所有数据本地存储，不依赖网络
2. 接口隔离——数据源通过接口注入，便于替换和测试
3. 功能渐进——从 MVP 基础记账逐步扩展至完整功能

两轮 Bug 修复（BUGFIX-001~002）共解决 27 个问题，
包括密码锁不生效、深色模式不持久、备份恢复异常、SnackBar 不自动消失等。
```

#### 选题原因

```
日常生活中许多人需要记账，但市面上的记账 App 功能冗余、
需要联网注册或含有广告。因此我们开发了"纯本地、轻量级、
无广告"的随手记。用户打开即用，无需注册账号。
```

#### 功能设置依据

```
各功能模块基于用户需求设计：
记账增删改查是核心功能；收入/支出分类区分资金流向；
月度汇总和预算让用户了解财务状况；
统计图表直观展示消费结构；CSV 导出支持数据迁移；
回收站防止误删；密码锁屏保护隐私。
```

#### 项目合规说明

```
本项目遵守我国法律和学校规章制度：
所有数据存储于本地，不收集个人信息；
不涉及网络请求；代码为小组成员原创。
```

#### 团队分工

```
在本次项目开发中，小组成员密切配合、各有侧重：

（成员A）负责记账核心功能（增删改查）和分类管理模块的编码实现。
（成员B）负责统计图表、CSV 导出、数据备份恢复模块的编码实现。
（成员C）负责附件上传、密码锁屏、深色模式模块的编码实现。
（成员D）负责搜索、回收站、UI 增强、月度汇总模块的编码实现。
吕加年负责系统架构设计、代码审查与测试、缺陷修复与验收、GitHub 仓库管理。

各模块开发完成后经代码审查合并，团队定期同步进度，
共同保障了项目的整体质量。
```
