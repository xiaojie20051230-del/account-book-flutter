# 随手记 期末项目 — 成员任务分工

## 怎么用这个文件夹

每个成员打开自己对应的文件夹，里面的 README.md 会告诉你**具体做什么、怎么交付**。

## 任务分配

| 任务 | 对应文件夹 | 交付物 | 交给谁 |
|------|----------|--------|--------|
| **A** AI 生成草图+流程图 | `A-实验目的与草图/` | `A-实验目的与草图.docx` | 发群里→D 合并 |
| **B** 搭环境跑 App+截图 | `B-截图与设计思路/` | `B-截图与设计思路.docx` | 发群里→D 合并 |
| **C** AI 生成总结与体会 | `C-总结与体会/` | `C-总结与体会.docx` | 发群里→D 合并 |
| **D** 合并成最终报告 | `D-合并报告/` | `创新实验-实验五-期末大作业-小组第3组.docx` | 提交 |
| **吕加年** 上传 GitHub | 直接做 | 仓库地址+截图+LICENSE | 发群里 |

## GitHub 上传注意事项（吕加年）

上传到 GitHub 时，**整个项目文件夹**包括：

```
02-期末综合项目/
├── LICENSE（MIT 许可证——已创建好了）
├── README.md
├── 需求分析与系统设计/    ← 全部设计文档（ADR/DESIGN/BUGFIX/TEST-PLAN/GUIDE）
├── 项目代码/             ← Flutter 源码
├── 测试记录/             ← 测试报告
├── 分工/                 ← 成员任务书
└── 实验报告/             ← 报告模板
```

**许可证必须加**，因为项目参考了其他开源 Flutter 项目，需要声明授权。

## 格式统一要求（所有人遵守）

- **字体**：宋体
- **字号**：5号
- **行距**：1.5倍行距
- **文件格式**：.docx

## 流程

```
吕加年发 GitHub 地址到群里
  → A/B/C 三人各自写 .docx → 发群里
    → D 下载 3 份 .docx → 合并成最终报告 → 发群里审核 → 提交
```
